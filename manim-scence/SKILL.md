# Manim Scence

## 目标层

### 输入预期

**用户初始请求通常模糊，如：**
- "帮我做一个关于微积分的视频"
- "傅里叶变换怎么讲才能让学生听懂？"
- "我想做个3Blue1Brown风格的数学动画"

**用户可能提供的细节：**
- 主题概念（必需）
- 目标受众（教育水平）
- 视频时长偏好
- 风格偏好（几何直观/代数推导）
- 特殊要求（必须包含的应用场景等）

**用户可能未明确但关键的信息：**
- 听众的先备知识
- 是否有特定"顿悟时刻"要强调
- 需要避免的内容

### 产出要求

**主产出物：**
- 一个完整的 `scenes.md` 文件
- 符合3Blue1Brown风格的分镜文档

**文档必须包含：**
- 视频概述（主题、钩子、受众、长度、关键见解、情感目标）
- 叙事弧线（三幕结构）
- 6-12个场景，每个场景包含：
  - 时长和情感目标
  - 卡片布局（带间距验证）
  - 卡片内容（元素描述+Manim代码示例）
  - 旁白示例（2-3句具体台词）
  - 过渡动画设计
- 配色方案
- 实现顺序建议
- 技术要点

**输出格式：**
- Markdown 文档
- 代码块使用 ```python``` 标记
- 表格格式呈现卡片布局

**参考模板：** [templates/scenes-template.md](templates/scenes-template.md)

## 知识层

### 3Blue1Brown风格核心特征

**视觉风格：**
- 深色背景（接近黑色 #1C1C1C）
- 鲜艳的配色（蓝 #4ECDC4、绿 #44A08D、橙 #FF6B6B）
- 流畅的动画过渡
- 几何图形优先于公式

**叙事风格：**
- 从具体问题出发，而非抽象定义
- 先建立困惑（钩子），再给出解答
- 强调"顿悟时刻"（Aha Moment）
- 旁白亲切、有启发性

**教育原则：**
- 多维度展示（代数、几何、物理意义）
- 由具体到抽象
- 视觉先行，直觉优先
- 预留思考停顿

### 数学教育视频结构知识

**标准叙事弧线：**
1. **第一幕：建立困惑**（场景1-2）
   - 问题引入：反直觉现象
   - 背景铺垫：统一知识基础
2. **第二幕：探索与发现**（场景3-5）
   - 核心概念多角度展示
   - 直观可视化解释
3. **第三幕：顿悟与升华**（场景6）
   - 核心洞见回顾
   - 应用场景展示
   - 启发思考

**场景时长分配：**
- 引入：45-60秒
- 铺垫：60-90秒
- 核心概念：90-120秒/个
- 总结：60-90秒

### 卡片布局系统

**画布规格：**
- 标准分辨率：14×8 单位（宽高比 16:9）
- 原点在中心 (0,0)
- 左右边界：±7
- 上下边界：±4

**布局模式：**
- **左右分屏（2卡片）**：可视化+说明
- **网格布局（4卡片）**：多角度对比
- **列表布局（3卡片）**：步骤展示
- **单卡片全屏**：核心概念强调

**间距计算：**
```
卡片A右边界 = center_x + width/2
卡片B左边界 = center_x - width/2
间距 = 卡片B左边界 - 卡片A右边界 ≥ 0.2
```

**参考：** [references/layout-examples.md](references/layout-examples.md) - 详细布局示例

## 行为层

### 思维链（7步工作流程）

**第1步：询问用户详细内容**

将模糊请求转化为明确需求。主动询问以下信息，并预设可能的回答供用户选择：

**必需信息：**
- **听众教育水平**："你的目标观众是什么水平？高中生、大学新生还是专业人士？"
- **视频时长**：提供选项（5-8分钟短视频、10-15分钟标准视频、20-30分钟长视频）
- **风格偏好**："偏重几何直觉还是代数推导？需要应用实例吗？"

**可选信息：**
- 特定的"顿悟时刻"或重点
- 历史背景或实际应用需求
- 需要避免的内容
- 配色偏好

**输出：**用户需求确认清单

**参考对话示例：** [references/examples.md#询问示例](references/examples.md)

---

**第2步：上网搜集相关知识**

**搜索维度：**
1. 数学推导：核心公式、定理证明
2. 直观可视化：几何解释、动画思路
3. 常见误区：学生易错点
4. 实际应用：工程/物理意义
5. 历史背景：概念起源

**输出格式：** 见 [references/examples.md#知识整理](references/examples.md)

---

**第3步：分场景（6-12个）**

**标准结构（最少6个）：**
1. **场景1：问题引入**（45-60秒）- 反直觉现象，建立好奇
2. **场景2：背景铺垫**（60-90秒）- 前置知识回顾
3. **场景3：核心概念（上）**（90-120秒）- 第一个视角
4. **场景4：核心概念（下）**（90-120秒）- 第二个视角
5. **场景5：直观理解**（90-120秒）- 几何可视化
6. **场景6：总结升华**（60-90秒）- 洞见回顾+应用

**可选扩展：**
- 场景7：实际应用
- 场景8：常见误区澄清
- 场景9：概念联系

**分配原则：**
- 每场景聚焦1-2个核心概念
- 场景间逻辑递进
- 每场景有情感目标（困惑→好奇→惊喜→满足）

**输出：**场景列表，包含名称、时长、情感目标

---

**第4步：分配知识给场景**

**筛选原则：**
1. 优先分配**适合可视化、能用几何动画表达**的知识点
2. 纯代数推导可简化或后置
3. 由浅入深，由具体到抽象
4. 多角度展示同一概念
5. 重要概念后预留停顿场景

**输出：**场景-知识点映射表

**参考示例：** [references/examples.md#知识分配](references/examples.md)

---

**第5步：布局设计**

**工作流程：**
1. **分析场景内容**：需要展示什么（图形、公式、文字、对比）？
2. **选择布局模式**：
   - 可视化+说明 → 左右分屏
   - 多角度对比 → 网格布局
   - 步骤展示 → 列表布局
   - 核心强调 → 单卡片全屏
3. **定义卡片**（必须验证不重叠）：使用表格定义卡片名、宽高、中心坐标、用途、颜色
4. **验证间距**：计算边界，确保间距 ≥ 0.2 单位
5. **卡片内设计**：
   - 文字：小号（24-36pt），位置灵活
   - 图形：可占满卡片80-90%
   - 公式：居中或顶部对齐

**输出：**每个场景的卡片布局定义

**参考示例：** [references/layout-examples.md](references/layout-examples.md)

---

**第6步：设计过渡动画**

**过渡类型：**
1. **内容延续**：上场景元素变换到下场景
2. **视角切换**：同一对象不同角度
3. **放大深入**：从overview到细节
4. **对比展示**：分屏对比

**过渡模板：** 见 [references/templates.md#过渡设计](references/templates.md)

**时序规范：**
- 场景间过渡：1-2秒
- 元素入场：0.5-1.5秒
- 强调动画：0.3-0.8秒
- 关键信息后停顿：0.5-1秒

**输出：**场景间过渡设计表

---

**第7步：生成scenes.md文档**

整合所有信息，生成完整文档（格式见协议层）。

---

### 工作原则

**内容筛选原则：**
- **可视化优先**：能用动画展示的优先，纯代数可后置
- **情感驱动**：每个场景必须有情感目标
- **留白艺术**：重要概念后预留消化时间

**布局设计原则：**
- **卡片不重叠**（首要原则）
- **铺满屏幕**：所有卡片宽度之和≈14，高度之和≈8
- **卡片内自由**：只要不越界，大小位置灵活

## 规范层

### 格式禁区

**文档结构必须包含：**
1. 视频概述（6要素）
2. 叙事弧线（三幕结构）
3. 6-12个完整场景
4. 配色方案
5. 实现顺序
6. 技术要点

**每个场景必须包含：**
- 时长（秒）
- 情感目标
- 卡片布局（带表格）
- 间距验证计算
- 卡片内容（含Manim代码）
- 旁白示例（2-3句）
- 过渡设计

### 数量禁区

**场景数量：**
- 最少：6个场景
- 最多：12个场景
- 违规后果：视频结构不完整或过于冗长

**卡片布局：**
- 必须验证间距 ≥ 0.2 单位
- 必须填满14×8画布（不留大片空白）
- 禁止卡片重叠（任何情况下）

### 内容禁区

**禁止：**
- 场景之间缺乏逻辑递进
- 场景没有明确的情感目标
- 缺少"顿悟时刻"设计
- 旁白示例少于2句或多于3句
- 卡片布局缺少间距验证计算
- Manim代码示例缺失或明显错误

### 思考模式禁区（Thinking Mode）

**自检时必须检查：**
- 场景数量是否在6-12范围内
- 每个场景的间距验证计算是否正确
- 是否有至少一个"顿悟时刻"
- 叙事弧线是否完整（困惑→探索→顿悟）

## 验证层

### 生成前自检（Thinking Mode）

**第1步完成后检查：**
- [ ] 已确认目标受众教育水平
- [ ] 已确认视频时长范围
- [ ] 已确认风格偏好（几何/代数）

**第3步完成后检查：**
- [ ] 场景数量在6-12范围内
- [ ] 每个场景有明确的情感目标
- [ ] 场景之间有逻辑递进关系
- [ ] 包含"问题引入"和"总结升华"

**第5步完成后检查（关键）：**
- [ ] 每个场景的卡片布局都有间距验证计算
- [ ] 所有卡片间距 ≥ 0.2 单位
- [ ] 卡片总宽度 ≈ 14（允许±0.5误差）
- [ ] 卡片总高度 ≈ 8（允许±0.5误差）

**第7步完成后检查：**
- [ ] 文档包含所有必需章节
- [ ] 每个场景都有旁白示例（2-3句）
- [ ] 有过渡动画设计
- [ ] 配色方案完整

### 质量检查清单

#### 结构完整性
- [ ] 最少6个场景，最多12个
- [ ] 每个场景有明确的情感目标
- [ ] 场景之间有逻辑递进
- [ ] 有过渡动画设计

#### 内容质量
- [ ] 搜集的知识点都已分配
- [ ] 每个场景有具体的旁白示例
- [ ] 有"顿悟时刻"的设计
- [ ] 包含实际应用或例子

#### 布局设计
- [ ] 使用了卡片式布局
- [ ] **卡片之间不重叠**（间距≥0.2）
- [ ] 所有卡片铺满14×8
- [ ] 每个卡片有透明调试边框（opacity=0.3）
- [ ] 文字大小合适（24-36pt）
- [ ] 图形大小合适（不越界）

#### 技术实现
- [ ] 有Manim代码示例
- [ ] 坐标计算正确
- [ ] 动画时序合理
- [ ] 配色方案完整

### 布局验证方法

**手动验证步骤：**
1. 计算每个卡片的左右上下边界
2. 检查任意两个卡片的水平间距是否 ≥ 0.2
3. 检查任意两个卡片的垂直间距是否 ≥ 0.2
4. 验证所有卡片宽度之和（含间距）≈ 14
5. 验证所有卡片高度之和（含间距）≈ 8

**Python验证脚本：** [references/validation-scripts.md](references/validation-scripts.md)

### 置信度评估

**生成前评估：**
- **高**：用户提供了详细信息，主题熟悉
- **中**：用户提供了一般信息，需要一些研究
- **低**：信息不足或主题不熟悉

**生成后评估：**
- **高**：所有检查项通过，布局合理，内容丰富
- **中**：大部分检查项通过，有小问题需要调整
- **低**：有明显缺陷，需要大幅修改

**低置信度处理：**
在文档开头添加注释：
```markdown
> ⚠️ **注意**：本方案置信度较低，建议在实现前与用户确认关键设计决策。
```
